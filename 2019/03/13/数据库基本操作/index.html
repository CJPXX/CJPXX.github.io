数据库基本操作

数据库的基本操作：

启动数据库:mysql -uroot -p;

退出数据库: quit;

1.数据库的基本操作

创建数据库 

CREATE DATABASE 数据库名字

查看数据库
SHOW DATABASE

查看某个已创建数据库信息 

SHOW CREATE DATABASE 数据库名称

修改数据库 修改数据库编码 

ALTER DATABASE xxgc
DEFAULT CHARACTER SET gbk COLLATE gbk_bin;

删除数据库 

DROP DATABASE 数据库名称(数据库不存在情况会报错) 
DROP DATABASE IF EXISTS 数据库名称(不存在情况下不会报错)

2.数据表的基本操作

1.创建数据表

USE 数据库名
CREATE TABLE 表名(
 字段名1 数据类型 [完整性约束条件]，
 字段名2 数据类型 [完整性约束条件]，
 ......
 字段名n 数据类型 [完整性约束条件]，
 );

2.查看数据表 

1. show create table 表名称
2. DESCRIBE 查看数据表名
3. DESC 表名

3.修改数据表

修改表名 

ALTER TABLE 被修改表名 RENAME TO 将修改成的表名；

修改字段名 

ALTER TABLE 表名 CHANGE 旧字段名 新字段名 新数据类型();

修改字段数据类型 

ALTER TABLE 表名 MODIFY 字段名 新数据类型();

添加字段 

ALTER TABLE 表名
 ADD 新字段名 数据类型 [约束条件][FIRST|AFTER 已存在字段名]

(方括号表示可选)

删除字段 

ALTER TABLE 表名 DROP 字段名;

修改字段的排列位置 

ALTER TABLE 表名 MODIFY 字段名1 数据类型 FIRST|AFTER 字段名2;

删除数据表 

DROP TABLE 表名

4.数据表约束

主键约束,用于唯一表示对应的记录 PRIMARY KEY

1.单字段主键 可以做列级约束,也可以做表级约束  

字段名 数据类型 PRIMARY KEY


2.多字段主键 只可作表级约束  

PRIMARY KEY(字段名1,字段名2,......字段名n)

外键约束 FOREIGN KEY

CONSTRAINT 自定义外键名 FOREIGN KEY(主表字段名) REFERENCES 副表名(副表字段名)

非空约束 NOT NULL

字段名 字段名 NOT NULL；

唯一性约束, 用于保证表中字段值不能重复出现 UNIQUE

字段名 数据类型 UNIQUE;

默认值约束，设置字段的默认值 DEFAULT 

字段名 数据类型 DEFAULT 默认值;

设置表的字段值自动增加（AUTO_INCREMENT约束的字段可以是任何整数类型，默认情况下从1开始自增）

字段名 数据类型 AUTP_INCREMENT;

5.数据字段插入

插入单条数据

1.向表中所有字段都插入数据

  INSERT [INTO] 表名(字段名1,字段名2,...) VALUES(值1,值2,...);

2.向表中部分字段插入一条记录

  insert into 表名(需要插入数据的字段名) VALUES(值);

插入多条数据

INSERT [into] 表名[(字段名1,字段名2,...,字段名n)] 
VALUES(值1,值2,...,值n),
             (值1,值2,...,值n),
             ...
             (值1,值2,...,值n);

6.修改和删除数据

修改数据表数据

UPDATE 表名 UPDATE  
   SET 列名=新数据值[...]
    [WHERE 子句];

   例如:    
   SET date='1998,05,11'
    WHERE name='张宇';

WHERE子句:指定要修改的记录的条件,可写可不写,如果不写,则表示将指定列下面的所有值都改成新的值;
若写了条件,则表示只将指定列下面满足条件的值改成新值。

删除数据表记录

DELETE FROM 表名                
[WHERE 子句];

WHERE子句:指定要删除的表的条件,可写可不写,
如果不指明条件,则表示清空指定的数据表,
若指明条件,则表示只将满足条件的记录删除

清空数据表

DELETE FROM 表名;
或  TRUNCATE[table] 表名;
即  TRUNCATE TABLE 表名;

delete语句是DML语句,TRUNCATE语句通常被认为是DDL语句
TRUNCATE语句比DELETE语句速度更快,而且使用系统和事物日志资源更少
